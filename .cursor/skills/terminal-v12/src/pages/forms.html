// Show button on input focus
const input = document.getElementById('orgInput');
const button = document.getElementById('orgBtn');

input.addEventListener('focus', () => {
  button.classList.add('show');
});

// Optional: hide on blur
input.addEventListener('blur', () => {
  setTimeout(() => {
    if (!button.matches(':hover')) {
      button.classList.remove('show');
    }
  }, 200);
});

14. Form Control Guidelines

Input Controls

  • Text Inputs - use .input class for all standard inputs and textareas
  • Form Groups - wrap label + input in .form-group for proper spacing (gap: 0.5rem)
  • Labels - always use .label class for form labels
  • Focus States - all inputs have blue ring focus state with box-shadow
  • Disabled Inputs - add disabled attribute for non-editable fields

Validation States

  • Error State - add .input--error class and .input-error message
  • Success State - add .input--success class and .input-success message
  • Message Styling - validation messages appear below inputs in red/green

Checkboxes & Radio Buttons

  • Checkbox Structure - use .checkbox__input, .checkbox__box, .checkbox__icon, .checkbox__label
  • Radio Structure - use .radio__input, .radio__circle, .radio__label
  • Mutual Exclusion - use radio buttons for single choice, checkboxes for multiple selections
  • Name Attribute - radio buttons in same group must share name attribute

Toggle Switches

  • Structure - use .toggle-label, .toggle-input, .toggle-switch, .toggle-text
  • Use Cases - binary on/off settings, not for mutually exclusive options
  • Active State - :checked state shows blue background with animated switch
  • Label Text - toggle text appears in uppercase automatically

Sliders & AI Inputs

  • Range Sliders - use .slider class with type="range"
  • Slider Labels - use .slider-label to display current value
  • AI Inputs - wrap in .input-ai-wrapper with .ai-indicator for AI features
  • AI Styling - AI inputs have dark background with pulsing gold indicator

Best Practices

  • Accessibility - always associate labels with inputs using proper structure
  • Placeholder Text - use placeholders for hints, not as replacements for labels
  • Error Prevention - validate on blur and show clear error messages
  • Mobile Friendly - all controls are touch-optimized with proper tap targets
  • Consistent Spacing - maintain uniform gaps between form elements using .form-group
